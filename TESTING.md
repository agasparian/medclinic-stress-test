# TESTING.md — Руководство тестировщика
## Стресс-тест маркетинга клиники / МедСпринт

---

## 1. Что это за проект

Telegram Mini App — квиз из 6 вопросов, который оценивает устойчивость маркетинга медицинской клиники. По итогам выдаёт балл 0–100, карту рисков, топ-3 проблемы и собирает контакты лида.

Состоит из двух независимых частей:
- **Mini App** (фронтенд) — статические файлы в `tg-app/`
- **Telegram Bot backend** — Vercel Serverless Function в `api/webhook.js`

## 3. Запуск без Telegram (браузер)

Никакой установки не требуется. Откройте в браузере:

```
https://medclinic-stress-test.vercel.app
```

**Или локально:** клонируйте репозиторий и откройте `tg-app/index.html` двойным кликом — всё работает без сервера.

**В браузере вместо нативных кнопок Telegram появляются DOM-заглушки:**
- Синяя фиксированная кнопка снизу (`#fallback-btn`) — заменяет Telegram `MainButton`
- Круглая кнопка `←` слева сверху (`#fallback-back-btn`) — заменяет Telegram `BackButton`
- Персональное приветствие (`Привет, Имя! 👋`) **не появится** — `tg.initDataUnsafe.user` пустой вне Telegram

```
[screen-welcome]    Стартовый
    ↓  кнопка «Начать стресс-тест»
[screen-question]   Вопросы Q1–Q6 (один динамический экран)
```

## 6. Тест-кейсы: стартовый экран

### TC-01 Счётчик клиник
**Действие:** Открыть приложение.
**Ожидание:** Счётчик анимируется от 0 до **146** за ~1200 мс (значение из `CONFIG.totalAudited`).

### TC-02 Оффер-модалка — первый визит
**Предусловие:** Очистить `localStorage` (DevTools → Application → Local Storage → Clear All).
**Действие:** Открыть приложение.
**Ожидание:** Появляется модальное окно с заголовком **«2 бесплатных инструмента для вашей клиники»** и тремя буллетами:
- `📊 Карта каналов: откуда приходили пациенты в 2025 (605 клиник)`
- `📍 Чеклист: 10 минут в Яндекс.Картах — канал №1 по звонкам`
- `🔔 Новые инструменты и обновления теста`

Кнопка: **«Получить бесплатно»**

### TC-03 Оффер-модалка — повторный визит
**Действие:** Перезагрузить страницу (не очищая `localStorage`).
**Ожидание:** Модалка **не появляется**. Ключ `medclinic_offer_shown` = `"1"` в `localStorage`.

### TC-04 Кнопка «Получить бесплатно» в оффере
**Действие:** Нажать «Получить бесплатно».
**Ожидание:**
- Модалка закрывается с анимацией (220 мс).
- В браузере открывается новая вкладка: `https://t.me/clinic_marketing_stress_test_bot?start=from_app`
- В `localStorage` появляется `medclinic_offer_shown = "1"`.

### TC-05 Кнопка «Пропустить» в оффере
**Действие:** Нажать «Пропустить».
**Ожидание:** Модалка закрывается. `medclinic_offer_shown = "1"` в `localStorage`.

### TC-06 Оффер на десктопе
**Действие:** Открыть в браузере с шириной ≥ 600px.
**Ожидание:** Модалка отображается по центру экрана, не приклеена к низу. Максимальная ширина карточки 420px, `border-radius: 20px` со всех сторон.

### TC-07 Оффер на мобильном
**Действие:** Открыть в браузере с шириной < 600px (или DevTools → мобильный эмулятор).
**Ожидание:** Модалка выезжает снизу, прикреплена к нижней части экрана. `border-radius: 20px 20px 0 0`.

## 7. Тест-кейсы: квиз

### TC-08 Прогресс-бар
**Действие:** Проходить вопросы по одному.
**Ожидание:**
- На Q1: бар пустой (0%), текст «Вопрос 1 из 6»
- На Q3 (если Q2 не пропущен): бар ~33%, текст «Вопрос 3 из 6»
- На последнем вопросе: бар не 100% (рассчитывается как `index/total`, а не `(index+1)/total`)

### TC-09 Q1 — множественный выбор
**Действие:** Нажать несколько вариантов в Q1.
**Ожидание:**
- Каждая карточка подсвечивается акцентным цветом при выборе, кружок заполняется галочкой.
- Повторное нажатие снимает выбор.
- Кнопка «Далее» активируется после выбора хотя бы одного варианта.

### TC-10 Q1 — эксклюзивный вариант «Не отслеживаем»
**Действие:** Сначала выбрать несколько каналов, потом нажать «❓ Не отслеживаем источники».
**Ожидание:** Все другие варианты снимаются, остаётся только «Не отслеживаем».
**Обратное:** Выбрать «Не отслеживаем», потом нажать любой другой канал.
**Ожидание:** «Не отслеживаем» снимается.

### TC-11 Пропуск Q2 при «Не отслеживаем»
**Действие:** В Q1 выбрать только «❓ Не отслеживаем», нажать «Далее».
**Ожидание:** Следующий вопрос — Q3 (SEO), Q2 пропущен. Текст: «Вопрос 2 из 5».

### TC-12 Q2 показывается при любых других ответах в Q1
**Действие:** В Q1 выбрать что угодно кроме «Не отслеживаем», нажать «Далее».
**Ожидание:** Q2 — «Какую долю маркетингового бюджета занимает платная реклама?». Текст: «Вопрос 2 из 6».

### TC-13 Single-select вопросы (Q2–Q6)
**Действие:** Выбрать один вариант, потом другой.
**Ожидание:** Предыдущий вариант снимается — активен только один.

### TC-14 Кнопка «Назад»
**Действие:** На Q3 нажать кнопку «←» (браузер) или BackButton (Telegram).
**Ожидание:** Возврат на Q2 с анимацией вправо, ранее выбранный ответ восстановлен.
На Q1 нажать «Назад» → возврат на стартовый экран.

### TC-15 Кнопка «Далее» заблокирована без ответа
**Действие:** Открыть любой вопрос, не выбирать ответ.
**Ожидание:** Кнопка серая / неактивная (`disabled`). Клик ничего не делает.

### TC-16 Последний вопрос
**Действие:** Дойти до последнего вопроса.
**Ожидание:** Кнопка называется **«Узнать результат»**, а не «Далее».

### TC-17 Сохранение прогресса
**Действие:** Пройти 3 вопроса, закрыть вкладку, открыть снова.
**Ожидание:** На стартовом экране появляется баннер:
`Есть незавершённый тест (шаг 4). [Продолжить] · [Начать заново]`
`localStorage` содержит `stress_test_answers` (JSON) и `stress_test_step` (число).

### TC-18 Продолжение незавершённого теста
**Действие:** Нажать «Продолжить» в баннере.
**Ожидание:** Квиз открывается на сохранённом вопросе с восстановленными ответами.

### TC-19 Сброс через «Начать заново»
**Действие:** Нажать «Начать заново» в баннере.
**Ожидание:** Баннер исчезает. `localStorage` очищается от `stress_test_answers` и `stress_test_step`.

| Ответ | Баллы |

### Уровни риска

| Балл | Уровень | Бейдж | Цвет |
|---|---|---|---|
| 75–100 | good | ХОРОШИЙ УРОВЕНЬ | зелёный #43A047 |
| 50–74 | moderate | УМЕРЕННЫЙ РИСК | оранжевый #FB8C00 |
| 25–49 | high | ВЫСОКИЙ РИСК | красный #E53935 |
| 0–24 | critical | КРИТИЧЕСКИЙ РИСК | тёмно-красный #B71C1C |

### TC-21 Верификация максимального балла
**Действие:** Q1: выбрать 5+ каналов, Q2: «Менее 30%», Q3: «Топ-5», Q4: «Активно», Q5: «МИС/CRM», Q6: «Сквозная аналитика».
**Ожидание:** Результат **100 / 100**, бейдж «ХОРОШИЙ УРОВЕНЬ», зелёный цвет.
Топ-3 проблем отсутствует, вместо них: «Серьёзных проблем не выявлено 👍»

### TC-22 Верификация минимального балла
**Действие:** Q1: «Не отслеживаем», Q3: «Нет в видимости», Q4: «Не занимались», Q5: «Базы нет», Q6: «Аналитики нет».
**Ожидание:** Результат **0 / 100**, бейдж «КРИТИЧЕСКИЙ РИСК», цвет #B71C1C.

### TC-23 Флаг BUDGET_RISK — сценарий риска
**Условие срабатывания:** Q2 = «Более 70%» или «Не считали», И в Q1 был выбран Директ (или Q1 не «Не отслеживаем»).
**Ожидание:** На экране результата появляется блок «⚡ Сценарий риска»:
`Директ подорожал на 30% → CPL: ~2 400 → ~3 100 ₽`

### TC-24 Флаги и их условия

| Флаг | Условие в коде |
|---|---|
| `CRITICAL` | Q1 = «Не отслеживаем» ИЛИ выбран только 1 канал |
| `BUDGET_RISK` | Q2 = `gt70`, `unknown` или `null` (пропущен) |
| `MONOCHANNEL` | Q1 содержит `direct`, но НЕ содержит ни одного из: `seo`, `maps`, `aggregators`, `tg` |
| `NO_ANALYTICS` | Q1 = «Не отслеживаем» ИЛИ Q6 = «Аналитики нет» |

### TC-25 Карта рисков — цвета полосок
**Ожидание (из кода `renderResult`):**
- % от макс. ≥ 70% → зелёный (#43A047)
- % от макс. 40–69% → оранжевый (#FB8C00)
- % от макс. < 40% → красный (#E53935)
- 0/max → иконка 🔴 справа
- < 50% от макс. → иконка ⚠️ справа

### TC-26 Сравнение со средним
**Ожидание:** Строка под бейджем:
`Средний по нише: 61/100 · Callibri, 605 клиник`

### TC-27 Топ-3 проблемы — порядок
**Алгоритм:** Блоки сортируются по убыванию `(max - score)`. Берутся 3 с наибольшим отставанием от максимума. Блок с максимальным баллом не попадает в список.

### TC-28 Кнопка «Поделиться результатом»
**Действие:** Нажать «🔗 Поделиться результатом».
**В Telegram:** Открывается диалог шеринга с текстом вида:
`Прошёл маркетинговый стресс-тест клиники — 38/100 (ВЫСОКИЙ РИСК). Проверьте свою клинику:`
URL: `https://t.me/clinic_marketing_stress_test_bot`
**В браузере:** Клик не вызывает видимого действия (ошибка в `try/catch` игнорируется — нормальное поведение).

## 10. Тест-кейсы: форма контактов

### TC-29 Автозаполнение имени
**В Telegram:** Поле «Имя» предзаполнено `first_name + last_name` из Telegram-аккаунта.
**В браузере:** Поле пустое.

### TC-30 Форматирование телефона
**Действие:** Ввести цифры в поле телефона.
**Ожидание:** Автоматическое форматирование в `+7 (XXX) XXX-XX-XX`.

| Ввод | Результат |
|---|---|
| `9991234567` | `+7 (999) 123-45-67` |
| `89991234567` | `+7 (999) 123-45-67` (8 заменяется на 7) |
| `79991234567` | `+7 (999) 123-45-67` |
| `123` | `+7 (123` |

### TC-31 Валидация формы — кнопка «Отправить»
**Ожидание:** Кнопка активна **только если** одновременно выполнены оба условия:
1. Телефон содержит ≥ 11 цифр
2. Чекбокс согласия отмечен

| Телефон | Согласие | Кнопка |
|---|---|---|
| пустой | нет | ❌ заблокирована |
| полный | нет | ❌ заблокирована |
| пустой | да | ❌ заблокирована |
| полный | да | ✅ активна |

### TC-32 Чекбокс согласия
**Действие:** Кликнуть по области согласия (не по ссылке).
**Ожидание:** Чекбокс становится синим с галочкой. Повторный клик снимает галочку.

### TC-33 Ссылка на политику конфиденциальности
**Действие:** Нажать «политикой конфиденциальности».
**Ожидание:** В Telegram — открывается `tg.openTelegramLink('https://example.com/privacy')`. В браузере — `preventDefault()` блокирует переход (ссылка `#example.com/privacy` — заглушка, не реальный URL).

### TC-34 Кнопка «Назад» из формы
**Действие:** На экране формы нажать «←».
**Ожидание:** Возврат на экран результата с анимацией вправо. Кнопка снова «Получить план усиления — бесплатно».

### TC-35 Отправка формы

## 16. Структура localStorage

| Ключ | Значение | Когда создаётся | Когда удаляется |
|---|---|---|---|
| `medclinic_offer_shown` | `"1"` | При нажатии «Получить бесплатно» или «Пропустить» | Только вручную |
| `stress_test_answers` | JSON объект ответов | При каждом выборе ответа | При переходе на loading |
| `stress_test_step` | Строка с номером шага | При каждом выборе ответа | При переходе на loading |

**Для сброса всего состояния:**
```javascript
localStorage.removeItem('medclinic_offer_shown');
localStorage.removeItem('stress_test_answers');
localStorage.removeItem('stress_test_step');
```

## 17. Как сообщать об ошибках

Для каждого найденного бага укажите:
1. **Окружение:** браузер (название + версия) / Telegram (версия) / ОС
2. **Экран:** ID экрана (`screen-welcome`, `screen-question`, etc.)
3. **Шаги воспроизведения:** нумерованный список
4. **Ожидаемое поведение**
5. **Фактическое поведение**
6. **Консоль браузера:** скопируйте ошибки из DevTools → Console
7. **localStorage** (если релевантно): DevTools → Application → Local Storage
