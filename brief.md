# Brief: «Стресс-тест маркетинга клиники»
## Telegram Mini App — MVP v1

**Продукт:** Квиз-аудит маркетинга медицинской клиники
**Цель:** Сбор контактов (имя + телефон) и продажа услуг агентства
**Аудитория:** Владельцы и маркетологи медицинских клиник
**Формат:** Telegram Mini App (WebApp), открывается из бота по кнопке
**Язык:** Русский

---

## Флоу приложения

```
[Бот] → кнопка «Пройти стресс-тест»
    ↓
[Экран 1] Старт
    ↓ кнопка «Начать»
[Экран 2] Вопрос 1 — Каналы (multi-select)
    ↓ кнопка «Далее»
[Экран 3] Вопрос 2 — Бюджет (single-select)
    ↓ кнопка «Далее»
[Экран 4] Вопрос 3 — SEO / органика (single-select)
    ↓ кнопка «Далее»
[Экран 5] Вопрос 4 — GEO / Карты (single-select)
    ↓ кнопка «Далее»
[Экран 6] Вопрос 5 — База пациентов (single-select)
    ↓ кнопка «Далее»
[Экран 7] Вопрос 6 — Аналитика (single-select)
    ↓ кнопка «Узнать результат»
[Экран 8] Загрузка (2–3 сек, без ввода)
    ↓ автопереход
[Экран 9] Результат
    ↓ кнопка «Получить план»
[Экран 10] Форма контактов
    ↓ кнопка «Отправить»
[Экран 11] Подтверждение
```

### Экраны 2–7 — Вопросы квиза

**Общая структура каждого экранa вопроса:**

**Верх (фиксированный):**
- Прогресс-бар: заполненная полоска + текст «Вопрос X из 6»
- Текст вопроса (крупный, 18px bold)
- Подсказка под вопросом (если есть, 13px серый курсив)

**Тело (скроллируемое при необходимости):**
- Карточки-ответы (full-width, 52px минимум, tap-target 44px+)
- Иконка или эмодзи слева от текста ответа

**Низ (фиксированный):**
- Telegram MainButton: «Далее» (активна только после выбора ответа)
- Telegram BackButton (нативный, вверху): возврат на предыдущий вопрос

**Поведение карточек:**
- Single-select: одна карточка подсвечивается, остальные гаснут
- Multi-select (Q1): карточки переключаются независимо, чекбокс-иконка внутри
- При выборе: `Telegram.WebApp.HapticFeedback.selectionChanged()`
- MainButton становится активной после выбора минимум одного ответа

**Что видит пользователь:**

- Бейдж уровня риска: 🔴 **ВЫСОКИЙ РИСК** (или 🟡 / 🟢 по порогу)

**Средний блок — Карта рисков:**
- Заголовок секции: «Карта рисков»
- 6 строк, каждая: иконка + название + мини-бар + балл + иконка статуса
  ```

  💰 Бюджет          ░░░░░░░░  0 / 20   🔴

  📍 GEO (Карты)     ████░░░░  8 / 15
  👥 База пациентов  ██░░░░░░  3 / 15   ⚠
  📊 Аналитика       ██░░░░░░  2 / 10   🔴
  ```

**Нижний блок — Топ-3 проблемы:**
- Заголовок: «Что исправить в первую очередь»
- 3 карточки, генерируются по наихудшим блокам:
  - Иконка ⚡ + одна строка конкретной проблемы на бизнес-языке
  - Пример: «Агрегаторы занимают вашу выдачу — вы платите за трафик, который мог быть бесплатным»

**Сценарий риска (если флаг BUDGET_RISK):**

  - «При росте ставок Директ ещё на 30%: CPL ~2 400 → ~3 100 ₽»

- Telegram MainButton (синяя): **«Получить план усиления — бесплатно»** → Экран 10
- Дополнительная кнопка под результатом: «Поделиться результатом» → `Telegram.WebApp.openTelegramLink()`

**Элементы:**
| Элемент | Описание |

| Бейдж уровня | Цветная плашка с иконкой и текстом |
| Карта рисков | 6 строк с мини-прогресс-барами (CSS, не SVG) |
| Топ-3 проблемы | Карточки, генерируются по баллам автоматически |
| Кнопка шаринга | Telegram share, текст с баллом и ссылкой на бота |
| BackButton | Скрыт (нет смысла возвращаться к вопросам) |

**Переходы:**

### Экран 10 — Форма контактов

**Что видит пользователь:**
- Заголовок: «Куда отправить детальный план?»
- Подзаголовок: «Эксперт свяжется в течение 2 часов и разберёт ваши точки роста»
- Поле «Имя» — автозаполнение из `Telegram.WebApp.initDataUnsafe.user.first_name`
- Поле «Телефон» — пустое, type=tel, маска +7 (XXX) XXX-XX-XX
- Поле «Название клиники» — опциональное, placeholder «Необязательно»
- Чекбокс: «Согласен с обработкой персональных данных» (ссылка на политику) — **обязателен по 152-ФЗ**
- Telegram MainButton: **«Отправить»** (активна только если телефон заполнен + чекбокс отмечен)

**Элементы:**
| Элемент | Описание |
|---------|----------|
| Поле «Имя» | Автозаполнение, редактируемое |
| Поле «Телефон» | type=tel, обязательное, маска +7 |
| Поле «Клиника» | type=text, опциональное |
| Чекбокс 152-ФЗ | Обязательный, ссылка на политику конфиденциальности |
| MainButton | Активна только при валидном телефоне + согласии |

**Данные, которые отправляются:**
```
- Имя (из формы)
- Телефон
- Название клиники (если заполнено)
- Итоговый балл
- Флаги риска (BUDGET_RISK, MONOCHANNEL, NO_ANALYTICS, CRITICAL)
- Баллы по каждому блоку
- Telegram user_id, first_name, username
- Временная метка
```

**После отправки:**
- Webhook → уведомление менеджеру в Telegram (имя + телефон + балл + флаги)
- Запись в Airtable / Google Sheets

**Переходы:**
- MainButton «Отправить» → Экран 11
- BackButton → Экран 9

## UI / Дизайн-требования

### Философия: нативный Telegram, не сайт

| Правило | Как реализовать |
|---------|-----------------|
| Использовать Telegram MainButton | Нижняя нативная кнопка, не свой footer |
| Использовать Telegram BackButton | Нативная кнопка «‹» в шапке |
| Тема пользователя | `var(--tg-theme-bg-color)`, `var(--tg-theme-text-color)` и т.д. |

## Архитектура навигации (FSM)

```
СТАРТ
  └── welcome → q1 → q2 → q3 → q4 → q5 → q6 → loading → result → contacts → done

Назад (BackButton):
  q2 ← q1
  q3 ← q2
  q4 ← q3
  q5 ← q4
  q6 ← q5
  (loading, result, done — BackButton скрыт)

Исключение: если в q1 выбрано «Не отслеживаем» →
  q2 пропускается (бюджет неизвестен) → переходим на q3
  Балл Q2 = 0, флаг NO_ANALYTICS
```
